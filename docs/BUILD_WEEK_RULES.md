PitchFlow's implementation and submission plan reflects the currently observed requirements:

- material use of Codex and GPT-5.6;
- one category, Developer Tools;
- a working consistently runnable project and repository URL;
- English public submission material;
- developer-tool installation, supported-platform, and judge test instructions;
- a public YouTube demonstration no longer than three minutes with audio;
- the primary build task's `/feedback` Session ID;
- transparent pre-existing/third-party material disclosure and rights; and
- free judge access through the judging period.

This file is a dated implementation checklist, not a substitute for the governing official pages.
