# PitchFlow demo script

Target duration: **2:35–2:50**. Hard maximum: **3:00**. Language: English. Narration: creator-authored script rendered with an AI-assisted macOS system voice, which the Build Week FAQ explicitly permits. No music or third-party footage.

The final edit must show only the real running PitchFlow product, creator-owned browser captures, verified exported assets, the public repository, and the redacted Codex run report. Product UI must never be reconstructed or invented for the video.

## 0:00–0:16 — the problem and promise

**Visual:** PitchFlow cover, then the live public viewer hero at `https://pitchflow-ten.vercel.app`.

**Narration:**

> Developers can finish a product while its launch story stays scattered across notes, generic AI copy, and disconnected exports. PitchFlow turns one pinned public GitHub repository into an evidence-linked launch system. Ship the code. PitchFlow ships the story.

## 0:16–0:38 — judge path first

**Visual:** Scroll the live unauthenticated viewer through the campaign, product captures, both motion masters, social system, and checksums.

**Narration:**

> This is the no-login judge path. It is a real PitchFlow dogfood campaign, cached immutably on Vercel. Judges can inspect the evidence, campaign, channel copy, real product captures, five-slide carousel, social graphics, landscape and portrait videos, asset hashes, and the complete ZIP without waiting for a model call.

## 0:38–1:05 — working local flow

**Visual:** One-command terminal launch, canonical repository input, pinned commit, and evidence workspace.

**Narration:**

> Fresh generation stays local. Run one command, paste a canonical public GitHub URL, and PitchFlow resolves a forty-character commit. It gathers bounded repository metadata, languages, tree records, documentation, manifests, and license evidence through fixed GitHub hosts. It never clones, installs, imports, builds, or executes the submitted project, and likely secrets are redacted before model use.

## 1:05–1:33 — material GPT-5.6 integration

**Visual:** Creative direction controls, redacted Codex report, campaign preview, and evidence links.

**Narration:**

> GPT-5.6 Sol is the material creative director, not a decorative chat box. Through the official Codex SDK and the developer's authenticated local Codex entitlement, it receives the bounded evidence and a strict JSON schema. The accepted dogfood turn used model GPT-5.6 Sol, required no repair, produced eight claims, and passed forty-nine evidence-link checks. Every factual surface points back to exact source records; unsupported metrics, customers, testimonials, and superlatives are rejected.

## 1:33–1:59 — edit once, export every channel

**Visual:** Claim review/edit state, capture provenance, verified handoff receipt, then a short excerpt from each accepted Remotion master.

**Narration:**

> The creator can refine a claim, which becomes explicitly user-supplied, or approve a supported inference. Two to four real product screenshots are validated locally and recorded with ownership and hashes. One manifest then drives the responsive microsite, exact-size social cards, carousel, channel copy, and deterministic Remotion masters in sixteen-by-nine and nine-by-sixteen, followed by a checksummed asset index and traversal-safe ZIP.

## 1:59–2:28 — how Codex built the product

**Visual:** Repository history, architecture diagram, passing tests, rejected media evidence, then the repaired handoff frame.

**Narration:**

> Codex was also the development partner throughout Build Week. It helped turn the finish line into executable contracts, implement the evidence engine, local SDK boundary, export pipeline, renderer, browser workspace, and security verifiers. The key product decisions remained explicit: local-first authentication, a public read-only viewer, pinned evidence, and no personal OAuth in the cloud. Independent review rejected two technically valid video candidates because their visible proof was weak. Codex repaired the content contract and reran the gates instead of weakening them.

## 2:28–2:47 — close

**Visual:** Live public viewer close, repository URL, public status showing generation disabled, then PitchFlow identity.

**Narration:**

> The result is a developer tool that makes launch claims inspectable, creative production reproducible, and the judge experience complete without a live presentation. Explore the viewer at pitchflow-ten dot vercel dot app, inspect the source at GitHub dot com slash sick n thirty-three slash pitchflow, and ship the story with the same rigor as the code.
